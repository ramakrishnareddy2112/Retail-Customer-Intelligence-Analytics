# Power BI Export Data Dictionary

All files are UTF-8 CSVs generated by `scripts/build_dashboard_exports.py` from
validated processed data and analytical reports. Dates use ISO `YYYY-MM-DD`
format. Monetary values are GBP.

## Semantic Rules

- **Completed sales:** positive quantity and price, excluding cancelled invoices.
- **Historical customer value:** identified completed-sales monetary value observed
  during the dataset period. It is not predictive CLV, margin, or profit.
- **Anonymous sales:** included in aggregate sales facts but not customer-level RFM,
  cohort, segment, or cluster analysis because no stable customer ID exists.
- **Returns/cancellations:** retained separately with signed and absolute values.
- **Outliers:** included in every export total. The validated processed data retains
  quantity, unit-price, and line-revenue flags; this export phase performs no
  deletion or arbitrary clipping.
- **Partial period:** December 2011 contains data only through 9 December and is
  flagged wherever the period appears.
- Percentage columns ending `_pct` store values from 0 to 100, not 0 to 1.

## `dim_date.csv`

**Grain:** one row per calendar date from 2009-12-01 through 2011-12-09.

| Column | Power BI type | Definition |
|---|---|---|
| `date` | Date | Unique calendar date; primary date key. |

## `fact_orders.csv`

**Grain:** one row per validated completed order. `order_id` is unique.

| Column | Power BI type | Definition |
|---|---|---|
| `order_id` | Text | Clean invoice/order identifier. |
| `order_date` | Date | Date of the first line timestamp for the order. |
| `month_start` | Date | First day of the order month. Convenience field; relate the fact to `dim_date` using `order_date`. |
| `customer_id` | Text | Clean customer identifier; blank for anonymous orders. Never convert to decimal. |
| `country` | Text | Order country from the validated order summary. |
| `order_revenue_gbp` | Decimal number | Sum of completed line revenue for the order. |
| `units_sold` | Whole number | Sum of positive completed-sale units in the order. |
| `distinct_products` | Whole number | Distinct stock codes in the order. |
| `line_count` | Whole number | Completed transaction-line count in the order. |
| `is_identified_customer` | True/False | True when the order has a usable customer ID. |

Flagged outlier lines remain inside order revenue, units, products, and line counts.
Use validated processed line data for line-level flag investigation; do not infer an
order is erroneous from its value alone.

## `dim_customer.csv`

**Grain:** one row per identified customer. `customer_id` is unique.

| Column | Power BI type | Definition |
|---|---|---|
| `customer_id` | Text | Clean customer identifier without decimal formatting. |
| `first_purchase_date` | Date | First observed completed purchase date. |
| `last_purchase_date` | Date | Last observed completed purchase date. |
| `recency_days` | Whole number | Days from 2011-12-10 snapshot to last purchase. Lower is more recent. |
| `frequency_orders` | Whole number | Distinct completed orders in the observed period. |
| `monetary_value_gbp` | Decimal number | Historical identified completed-sales value. |
| `average_order_value_gbp` | Decimal number | Historical monetary value divided by frequency. |
| `units_purchased` | Whole number | Completed units associated with the customer. |
| `distinct_products` | Whole number | Distinct purchased stock codes. |
| `is_repeat_customer` | True/False | True when frequency is at least two completed orders. |
| `r_score` | Whole number | Recency quantile score, 1–5; 5 is best. |
| `f_score` | Whole number | Frequency quantile score, 1–5; 5 is best. |
| `m_score` | Whole number | Monetary quantile score, 1–5; 5 is best. |
| `rfm_score` | Whole number | Sum of R, F, and M scores. |
| `rfm_segment` | Text | One of the eight validated ordered rule-based segments. |
| `cluster_id` | Whole number | Final operational K=4 cluster ID, 1–4. |
| `cluster_label` | Text | Validated concise K=4 business label. |

Valid cluster labels are:

1. High-Value Champions
2. Recent Growth Customers
3. Lapsed Established Customers
4. Dormant Low-Value Customers

Clusters are descriptive. They do not predict churn, future value, or campaign
response and do not establish causality.

| `month_start` | Date | First day of the sales month. |

## Additivity Guidance

- Revenue and units are additive across their fact-table grains.
- Distinct orders/customers are not additive across product, country, or month
  groups; use the supplied measures or distinct counts at query time.
- Customer/value shares should sum to approximately 100% within their summary
  table, subject only to display rounding.
- Do not sum `cohort_size` across cohort-month rows; it repeats for every observed
  month of the same acquisition cohort.
